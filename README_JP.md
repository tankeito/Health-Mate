# 🏥 Health-Mate

> OpenClaw 向けのローカル優先ヘルスレポート技能。
>
> 構造化 Markdown の健康記録を、日報・週報・月報 PDF と任意の文字版配信に変換します。

## ✨ 概要

Health-Mate は、習慣トラッカーと慢性疾患セルフマネジメントの中間にあるレポート指向ツールです。

- 🧠 疾患対応: 胆石症 / 慢性胆嚢炎、高血圧、糖代謝管理、減量、複数疾患の同時管理に対応
- 📄 レポート重視: 日報・週報・月報を中心に設計
- 🔒 ローカル優先: 解析、採点、回退ロジック、PDF 描画は既定でローカル実行
- 🧩 拡張可能: 服薬、血圧、血糖、生化学、体脂肪、睡眠などの独自モジュールを追加可能
- 🌏 `1.4.1`: 日本語ドキュメントと `assets/NotoSansJP-VF.ttf` を追加

---

## ⚙️ 動作の流れ

1. OpenClaw が `MEMORY_DIR` に構造化 Markdown を保存
2. Python が食事、水分、体重、症状、服薬、運動、歩数、独自モジュールを解析
3. 疾患条件と複数条件の組み合わせに応じて採点
4. ローカル LLM を優先し、失敗時はローカルルール、必要に応じて Tavily を併用して PDF と文字版を生成

---

## 🧭 AI 出力ソース

| ラベル | 意味 |
| --- | --- |
| `Source: dynamically generated by LLM` | ローカル `openclaw agent --local` が成功 |
| `Source: Tavily retrieval + local rules` | ローカル LLM が失敗し、Tavily 補強付き回退を使用 |
| `Source: local rules` | ローカル LLM が失敗し、純粋なローカル回退を使用 |

補足:

- 定時実行は shell 環境で走るため、手動 OpenClaw 実行と結果が異なることがあります
- shell 側でローカル LLM が使えない場合、回退モードになります

## 🚀 クイックスタート

### 1. インストール

```bash
git clone https://github.com/tankeito/Health-Mate.git health-mate
cd health-mate
pip install -r requirements.txt
```

### 2. 環境変数

最低限:

```bash
MEMORY_DIR="/absolute/path/to/health-memory"
```

任意:

```bash
TAVILY_API_KEY="tvly-..."
DINGTALK_WEBHOOK="https://..."
FEISHU_WEBHOOK="https://..."
TELEGRAM_BOT_TOKEN="..."
TELEGRAM_CHAT_ID="..."
REPORT_WEB_DIR="/var/www/html/reports"
REPORT_BASE_URL="https://example.com/reports"
ALLOW_RUNTIME_FONT_DOWNLOAD="false"
```

### 3. 初期設定

```bash
python scripts/init_config.py
```

このウィザードは `user_config.json` の主要項目をカバーします。

- 基本プロフィール
- 複数疾患設定
- 水分・歩数目標
- 居住地
- AI モード
- 採点モジュールと重み
- 服薬設定
- カスタム監視モジュール
- Tavily 設定
- レポート設定

### 4. レポート生成

```bash
python scripts/daily_health_report_pro.py /path/to/memory/2026-03-20.md 2026-03-20
python scripts/weekly_report_pro.py 2026-03-20
python scripts/monthly_report_pro.py 2026-03-20
```

### 5. shell ランナー

```bash
scripts/daily_health_report_pro.sh
scripts/weekly_health_report_pro.sh
scripts/monthly_health_report_pro.sh
```

## 📄 レポート構成

### 日報

- 総合スコア表
- 栄養ドーナツ図
- 飲水・運動詳細
- AI コメント
- リスク警告

## 🔐 セキュリティと運用メモ

- `MEMORY_DIR` は必須で、明示的に設定する必要があります
- shell ランナーは `config/.env` を読み込みます
- レポートは `reports/`、ログは `logs/` に書き込まれます
- Tavily、Webhook、実行時フォントダウンロードは、対応する設定がある場合のみ外部通信します
- 更新や再インストール前に、`config/user_config.json` と `config/.env` を必ずバックアップしてください
