# Health-Mate

Health-Mate is an executable OpenClaw skill, not a prompt-only package.

It reads structured Markdown logs from `MEMORY_DIR`, generates Daily / Weekly / Monthly PDF reports, and can optionally deliver the text summary to DingTalk, Feishu, or Telegram.

## What It Does

- parses meals, hydration, exercise, symptoms, medication, steps, and custom monitoring sections
- generates daily reports with scoring, charts, AI commentary, risk alerts, and next-day plans
- generates weekly reports with heatmaps, trends, badges, review text, and next-week plans
- generates monthly reports with radar charts, healthy-day heatmaps, specialty graphs, follow-up reminders, and hospital / doctor planning
- supports multi-condition management in both LLM and fallback paths
- keeps parsing, scoring, and PDF rendering local by default

## Local And Network Behavior

Expected local file I/O:

- reads Markdown logs from `MEMORY_DIR`
- reads `config/.env` when bundled shell runners are used
- writes PDFs into `reports/`
- writes logs into `logs/`
- may create a temporary English memory mirror for font fallback rendering

Expected network I/O:

- Tavily only when `TAVILY_API_KEY` is configured
- webhook delivery only when the corresponding webhook settings are configured
- runtime font download only when `ALLOW_RUNTIME_FONT_DOWNLOAD=true`

Important:

- there is no implicit fallback `MEMORY_DIR`
- the shell runners exit if `MEMORY_DIR` is missing
- the shell runners read `config/.env`, so any secret placed there becomes readable to the skill runtime

## Runtime Entry Points

- `python scripts/daily_health_report_pro.py <memory_file> <date>`
- `python scripts/weekly_report_pro.py <YYYY-MM-DD>`
- `python scripts/monthly_report_pro.py <YYYY-MM-DD>`
- `scripts/daily_health_report_pro.sh`
- `scripts/weekly_health_report_pro.sh`
- `scripts/monthly_health_report_pro.sh`

## Upgrade And Backup Notice

Before upgrading or reinstalling this skill, back up:

- `config/user_config.json`
- `config/.env`
- any font files you manually placed into `assets/`

Important:

- some platform upgrade or reinstall flows may overwrite, reset, or remove local configuration files
- after upgrading, re-check `MEMORY_DIR`, scoring modules, Tavily settings, webhook settings, and report preferences before re-enabling scheduled jobs

## AI Source Labels

Health-Mate uses the following source labels:

- `Source: dynamically generated by LLM`
  Local `openclaw agent --local` succeeded.

- `Source: Tavily retrieval + local rules`
  Local LLM failed or was unavailable, and Tavily-enriched fallback was used.

- `Source: local rules`
  Local LLM failed or was unavailable, and pure local fallback was used.

Why scheduled jobs can differ from manual OpenClaw testing:

- scheduled jobs run in the shell environment loaded from `config/.env`
- manual OpenClaw tests may run in a richer interactive environment
- if the shell job cannot run the local LLM successfully, the report will downgrade to fallback mode

## Memory Write Protocol

When writing into `MEMORY_DIR`, the model must behave like a strict recorder.

Hard rules:

1. Never write commentary, advice, summaries, emoji, or chat filler into the memory file.
2. Meals, hydration, medication events, and exercise events must use level-3 headings with time markers.
3. Hydration blocks must remain minimal and structured.
4. Step totals must stay in one dedicated level-2 block.
5. Monitoring modules must use stable level-2 headings.
6. Use one language per block.

English core template:

```markdown
# 2026-03-20 Health Log

## Meals
### Breakfast (around 08:30)
- Oatmeal 50g -> approx. 190kcal
- Skim milk 250ml -> approx. 87kcal

## Hydration
### Morning (around 09:45)
- Water intake: 300ml
- Cumulative: 300ml/2000ml

## Exercise
### Afternoon Cycling (around 17:17)
- Distance: 10km
- Duration: 47min
- Burn: approx. 300kcal

## Today Steps
- Total steps: 8500 steps
```

Chinese core template:

```markdown
# 2026-03-20 健康记录

## 饮食记录
### 早餐（约 08:30）
- 燕麦片 50g -> 约 190kcal
- 脱脂牛奶 250ml -> 约 87kcal

## 饮水记录
### 上午（约 09:45）
- 饮水量：300ml
- 累计：300ml/2000ml

## 运动记录
### 下午骑行（约 17:17）
- 距离：10km
- 耗时：47分钟
- 消耗：约 300kcal

## 今日步数
- 总步数：8500 步

## 用药记录
- 胆舒胶囊：1 粒
```

Chinese monitoring-module constraints:

- `## 血压记录`、`## 血糖记录`、`## 体成分`、`## 生化情况` 这类标题必须保持稳定
- 数值型记录请使用稳定标签，例如 `- 血压：128/82 mmHg`、`- 血糖：7.1 mmol/L`、`- ALT：34 U/L`
- 监控模块中禁止写点评、总结、状态说明
- 不要把监控模块改写成表格
- 不要在不同日期随意改名同一个模块

Expandable monitoring examples:

```markdown
## Blood Pressure
### Morning (around 08:00)
- Blood Pressure: 128/82 mmHg
- Heart Rate: 72 bpm

## Glucose Record
### After Breakfast (around 10:10)
- Glucose: 7.1 mmol/L
- Timing: 2h after breakfast

## Biochemistry
- ALT: 34 U/L
- AST: 28 U/L
```

Recommended LLM instruction:

- paste the English or Chinese template above directly into your system prompt, `soul.md`, or memory-write policy
- if custom modules are enabled in `user_config.json`, add their exact section titles into the write protocol and keep them stable across days

Forbidden content:

- `Assessment`
- `Status`
- `Summary`
- emoji-only status lines
- extra prose outside the strict record structure
